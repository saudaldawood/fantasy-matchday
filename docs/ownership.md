# Ownership Model

This project uses role-based ownership so each member has a clear technical domain to understand, maintain, review, and explain.

| Member | Primary Ownership | Files and Concepts |
| --- | --- | --- |
| Bakr Jamjoom | Git/project management, deployment, documentation | Repository workflow, Firebase deployment config, setup docs, release checklist |
| Khalid Aleissa | Frontend/UI | Next.js pages, UI components, CSS Modules, responsive layouts, user-facing flows |
| Saud Dawood | Cloud Functions/backend | Firebase Functions, scheduled jobs, API-Football sync, notifications, admin functions |
| Fahad Alhomaidhi | Authentication/database/security | Firebase Auth, Firestore data model, rules, indexes, protected data |
| Abdulmohsen Binkhamis | Leaderboard/business logic | Scoring rules, lineups, leaderboard updates, credits, achievements |

Ownership means accountability for comprehension and review. It does not prevent teammates from contributing to the same files when needed.
